# Food Delivery

Сервис заказа готовой еды: пользователь выбирает готовые наборы и отдельные блюда, оформляет доставку и прикладывает чек о переводе. Сотрудники управляют меню и заказами через Filament.

## Статус

Проект находится на этапе создания MVP.

## Технологии

- Laravel 13 — бизнес-логика, HTTP API, хранение заказов.
- Vue 3 + Inertia.js — клиентская витрина без отдельного API-проекта.
- Filament 5 — административная панель.
- PostgreSQL 18 — основная база данных.
- Redis 8 — кеш и очереди.
- Tailwind CSS 4 + Vite 8 — стили и сборка frontend.
- Docker Compose — локальная разработка и одинаковое окружение команды.

## Возможности MVP

- Меню на следующий день с отдельной публикацией блюд и наборов по дате.
- Состав набора из нескольких блюд.
- Состав отдельного блюда выводится в карточке и быстром просмотре.
- Корзина с наборами и дополнительными товарами.
- В карточках каталога после первого добавления кнопка меняется на степпер с `+ / - / количество`.
- Оформление заказа без обязательной регистрации.
- Телефон заказа с российской маской и серверной нормализацией.
- Обязательный Telegram-ник в заказе и профиле с автосохранением для пользователя.
- Загрузка изображения или PDF-чека.
- Автоматическая доставка на завтра с 9:00 до 12:00 МСК.
- Серверная проверка дедлайна приёма заказов из настроек сайта.
- Фиксация цен и названий внутри заказа для сохранения истории.
- Фиксация состава отдельного блюда внутри строки заказа.
- Статусы заказа и оплаты.
- Управление товарами, наборами и заказами в `/admin`.
- Управление баннерами главного экрана в `/admin`.
- Управление стоимостью доставки, порогом бесплатной доставки, реквизитами оплаты и текстами checkout в разделе `Настройки сайта`.
- Для баннеров доступны загрузка файла, внешний URL картинки и отдельная ссылка перехода.
- Для баннеров доступны опциональные переиспользуемые теги, которые можно выбирать, создавать или не использовать.
- Для блюд и наборов доступны общие переиспользуемые теги каталога, например `Новинка`.
- Публичный раздел новостей с отдельными страницами и выводом на главной.
- Единые хлебные крошки на внутренних страницах сайта.
- Отдельная страница контактов и управляемое меню сайта через админку.
- Глобальные toast-уведомления для добавления в корзину и системных действий вроде входа, регистрации и оформления заказа.
- Публичная витрина на одной Inertia-странице с checkout-формой.
- Hero-блок главной страницы с адаптивным баннер-слайдером.
- На сайт добавлен общий футер с навигацией и контактами.
- Быстрый просмотр блюда или набора во всплывающем окне без перехода на отдельную страницу.
- Расширенное тестовое меню с реальными stock-изображениями для всех карточек.
- Для моков каталога используются внешние stock-изображения из открытых фотостоков.
- Регистрация, вход и личный кабинет.
- Вход и регистрация открываются в глобальном modal-окне поверх текущей страницы с переключением режима внутри формы и без редиректа в личный кабинет.
- История заказов и редактирование профиля пользователя.
- Повтор заказа из личного кабинета с проверкой актуальной доступности позиций.

## Модель предметной области

- `categories` — категории меню.
- `products` — отдельные блюда и напитки.
- `meal_sets` — готовые наборы.
- `meal_set_items` — состав набора и количество каждого блюда.
- `orders` — контакты, адрес, доставка, суммы, статусы и чек.
- `order_items` — строки заказа типа `product` или `meal_set`.
- `order_item_components` — снимок состава заказанного набора.

У блюд и наборов есть `menu_dates`: в админке можно указать несколько дат меню, а публичная витрина показывает позиции, доступные на завтра. Если у отдельного блюда список дат пуст, оно считается постоянным и доступно для заказа каждый день. Для набора необходимо указать хотя бы одну дату. Все денежные значения хранятся целыми копейками. Заказ хранит снимок названия, цены и состава: изменение меню не переписывает старые заказы.

## Деплой на сервер

Проект деплоится через GitHub Actions при push в ветку `main`.

### Сервер

- репозиторий на сервере: `/var/www/timetoeat`
- Docker Compose: `compose.yaml` + `compose.prod.yaml`
- сайт слушает порт `80`

Первичная настройка сервера:

```bash
bash scripts/server-init.sh
```

После этого отредактируйте `/var/www/timetoeat/.env` (скопирован из `.env.production.example`): задайте `APP_KEY`, пароль БД, `APP_URL`, токены Telegram/MAX.

Первый и последующие деплои на сервере:

```bash
cd /var/www/timetoeat
bash scripts/deploy.sh
```

### GitHub Secrets

В репозитории GitHub → Settings → Secrets and variables → Actions добавьте:

| Secret | Значение |
| --- | --- |
| `SERVER_HOST` | IP или домен сервера |
| `SERVER_USER` | `root` или отдельный deploy-пользователь |
| `SERVER_SSH_KEY` | приватный SSH-ключ для деплоя |

Публичный ключ деплоя должен быть в `~/.ssh/authorized_keys` на сервере.

Быстрая настройка секретов (после `gh auth login`):

```bash
bash scripts/set-github-secrets.sh
```

### CI/CD

Workflow `.github/workflows/deploy.yml` подключается по SSH и запускает `scripts/deploy.sh`: `git pull`, сборка frontend, миграции, кеш Laravel, перезапуск контейнеров.

## Запуск

Требуется установленный Docker Desktop.

```bash
make init
```

После запуска:

- сайт: http://localhost:8088
- админ-панель: http://localhost:8088/admin
- Vite: http://localhost:5174

Данные администратора для локальной разработки:

- email: `admin@example.com`
- пароль: `password`

Полезные команды:

```bash
make up
make down
make migrate
make fresh
make test
make lint
make shell
```
